ü§ñ CI/CD Integration

Laravel Migration Linter integrates seamlessly with GitHub Actions, GitLab CI, and other continuous integration pipelines.
It ensures no unsafe schema changes reach production by failing builds when risky migrations are detected.

üß∞ GitHub Actions Example‚Äã

name: Laravel Migration Linter

on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]

jobs:
  lint-migrations:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: shivammathur/setup-php@v2
        with:
          php-version: 8.3
          extensions: mbstring, pdo, bcmath, curl, dom, xml
          tools: composer:v2

      # üß© Example: setting up a test Laravel project using your local package
      - run: composer create-project laravel/laravel test-laravel-app --quiet

      - name: Install Laravel Migration Linter (local path)
        run: |
          cd test-laravel-app
          composer config repositories.sufyandev path ../
          composer require sufyandev/laravel-migration-linter:@dev --no-interaction

      - name: Run Migration Linter
        run: |
          cd test-laravel-app
          php artisan migrate:lint --json > lint-report.json

üìä Exit Codes and Build Behavior‚Äã

Severity ThresholdDescriptionExit CodeBuild Result
infoAdvisory messages only0‚úÖ Pass
warningPotential issues found, not fatal0‚úÖ Pass
errorMigration-breaking or data-loss risks1‚ùå Fail

You can adjust this threshold via config/migration-linter.php:

'severity_threshold' => 'error',

When set to 'error', only error-level issues cause your CI pipeline to fail.

üß† Tips for CI/CD Integration‚Äã

  • üíæ Commit the baseline file (migration-linter-baseline.json) once generated.
  • This prevents known legacy issues from reappearing in reports.
  • üß© Use --json output to generate structured reports for custom dashboards.
  • You can also upload the JSON artifact for analysis.
  • üö¶ Fail fast: set 'severity_threshold' => 'error' in config to stop deployments on risky migrations.
  • üîÅ Run on every PR ‚Äî this ensures every migration is linted before merge.

üßæ GitLab CI Example‚Äã

If you're using GitLab, add a stage like this:

  - lint

lint_migrations:
  stage: lint
  image: php:8.3
  script:
    - apt-get update && apt-get install -y unzip git
    - curl -sS https://getcomposer.org/installer | php
    - php composer.phar install --no-interaction
    - php artisan migrate:lint --json > lint-report.json

‚úÖ Result: Your CI/CD pipeline now automatically enforces safe, production-ready migrations ‚Äî blocking schema-breaking changes before they reach your main branch.
